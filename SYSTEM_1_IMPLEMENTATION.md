### 6. Price Monitor Agent (`src/app/api/agent/price-monitor/route.ts`)

**This is the CORE intelligence agent. It:**

✅ **Step 0:** Verifies Amadeus and Supabase are configured (helpful error messages if not)

✅ **Step 1:** Creates agent run log in `agent_runs` table

✅ **Step 2:** Fetches flight prices for all 42 monitored routes:
- For each of 6 source cities:
  - Searches flights to 2 Goa airports (GOI, GOX)
  - Searches flights to 5 competitor destinations (BKK, HKT, SGN, DPS, CMB)
- Uses Amadeus API `searchFlightOffers()` endpoint
- Searches 30 days out (next month departure)
- Gets top 3 cheapest flights per route

✅ **Step 3:** Stores ALL results in `flight_prices` table:
- Origin/destination details
- Price in original currency + converted to INR
- Airline, direct flight flag, cabin class
- Full raw API response (JSONB) for debugging

✅ **Step 4:** Analyzes the data — compares Goa vs competitors:
- For each origin city, calculates average price to Goa vs competitors
- If Goa is >1.5x more expensive, generates an analysis
- Severity levels:
  - **Critical** (red): >3x more expensive
  - **Warning** (amber): 2-3x more expensive
  - **Info**: 1.5-2x more expensive
- Includes actionable recommendations

✅ **Step 5:** Stores insights in `agent_analyses` table:
- Analysis type: `price_comparison`
- Title, summary, detailed breakdown
- Confidence score (0.9 for price comparisons)
- Recommended actions
- Expires in 7 days (so stale data doesn't accumulate)

✅ **Step 6:** Updates agent run log with results:
- Status: `completed`, `partial`, or `failed`
- Metrics: routes checked, records inserted, analyses generated
- Error messages if any routes failed

✅ **Step 7:** Returns JSON summary for monitoring

**Rate Limiting:**
- 500ms delay between API calls (to be nice to Amadeus)
- Well within free tier: 42 routes/day = 1,260 requests/month (limit: 2,000)
